Notes & security/UX recommendations

Sending plaintext passwords by email is insecure — it's implemented per your request to allow admin to share credentials, but recommend:
Create a password-reset link flow (tokenized), or force password change on first login.
Use secure SMTP (PHPMailer) to avoid mail acceptance issues.
CSRF protection: not yet added — I recommend adding CSRF tokens to all forms next.
Email deliverability: your PHP mail() needs a working MTA (XAMPP on Windows usually requires external SMTP relay); I can add PHPMailer and sample SMTP config so you can use Mailgun/SendGrid/Gmail SMTP.
UI/UX: added animations; we can further improve with a small icon set, better spacing, loader states, and more consistent alerts. I can produce a short UI polish pass.
